# my-page-test

#### ディレクトリ構成
1. 仕様書ファイル（docs/ 内）
一巨大なファイルにするより、役割ごとに分けるとAIが読み込みやすくなります（トークン節約にもなります）。

system_design.md（システム設計書）

内容: 「何を作るのか」「技術スタック（Python, Streamlit, Dockerなど）」「データベース構造（テーブル定義など）」「APIの仕様」。

理由: AIにコンテキスト（前提条件）を理解させ、技術選定のブレをなくすため。

screen_spec.md（画面・機能仕様書）

内容: 「画面一覧」「各画面のUI要素」「ボタンを押したときの挙動」。

理由: 「そんな画面聞いてない」というAIのサボりや勘違いを防ぐため。

todo.md（タスクリスト）

内容: 実装すべきタスクの箇条書き。

理由: AIに「次はこれを作って、終わったらチェックを入れて」と指示することで、開発の進捗をAI自身に管理させることができます。

2. プロジェクトの入り口（ルート直下）
README.md

内容: アプリの起動方法や、環境構築の手順。

理由: AIが自分で環境を立ち上げてテストしたり、エラーを自己解決したりする際の基準になります。

.cursorrules などのAIルールファイル

内容: 「コードコメントは日本語で」「テストコードを必ず書くこと」「絶対に使用してはいけないライブラリ」などの共通ルール。

理由: 開発中のすべてのチャットで、AIに毎回同じ前提（ルール）を強制的に守らせるために必須です（CursorやClineなどのAIエディタを使う場合に有効）。
