# 画面仕様書 (v2.4)

> **更新日**: 2026年6月28日  
> **ベース**: `画面仕様書 (v2.3)` + 追加要件（2026/06/28・`docs/202606_spec5/direction.md`）  
> **実装範囲**: 本仕様の追加要件はすべて**フロントエンド完結**。新規バックエンドAPI / スキーマ変更は不要。
> - 「本日」の判定はクライアントのシステム日付と各セルの日付（`daily_activity[].date` / `daily_status[].date`）の突合で行う。
> - セレブレーション演出の発火条件（OR）は、既存 `GET /api/progress` の `daily_status`（今週の達成日数 = 7）と `artist_date`（実施済み）を再利用して判定する。

## 2. 各画面詳細仕様（v2.4 追加・変更分）

### 2.1 統計画面（改修）
- **パス**: `/statistics`

#### ① ヒートマップへの現在日付（本日）の表示
- 日次アクティビティヒートマップ（12週×7日＝84マス）上で、**現在の日付（本日）のセルを視覚的に特定できる**ようにする。
- **判定**: クライアントのシステム日付（ローカル）と各セルの `daily_activity[].date` を突合し、一致するセルを「本日」とする。
- **表示方法（本バージョンの方針）**: 本日セルの**枠線をアクセントカラーで強調**する（リング）。あわせて、本日セルのホバー時ツールチップに「**今日**」を明記する。塗りつぶし色（文字数に応じた濃淡）は従来通りで、本日の識別は枠線で行う（記録の有無に関わらず本日が分かる）。
- 本日がプログラム期間（84日）外の場合は、いずれのセルも「本日」として強調しない。

#### レイアウト（参考）
```
┌────────────────────────────────────────┐
│ 日次アクティビティヒートマップ           │
│ (84マス: 12週×7日のGitHub草風)         │
│  □ □ ■ ■ ▣ □ □   ← ▣ = 本日(枠線アクセント)│
│  hover: "今日 / 680文字"                │
└────────────────────────────────────────┘
```

### 2.2 ユーザーホーム画面（改修）
- **パス**: `/home`

#### ① 「今週の実行状況」における本日の強調
- 「今週の実行状況」を表示するセクション（7日間のドットグリッド）において、**現在の日付（本日）に該当する曜日・日付の要素をハイライト**する。
- **判定**: クライアントのシステム日付と `daily_status[].date` を突合し、一致する Day を「本日」とする。
- **表示方法（本バージョンの方針）**: 本日のドット/セルに**「今日」を示す小さなバッジ**を付与し、あわせて**枠線（リング）をプライマリカラー**で強調する。達成状態（実行済み/未実行）の色表現とは独立して重畳する。
- ※ 過去週・未来週を閲覧中（タイムトラベル中）は、その週に本日が含まれないため強調は表示されない（現在週を表示している場合のみ本日が出現する）。

#### ② 過去週の「(閲覧のみ)」テキスト削除
- 過去の週のデータを表示している際に表示していた「**(閲覧のみ)**」というテキストを**削除**し、UIをクリーンに保つ。
- ※ 入力ボタンの非活性（閲覧のみの挙動）自体は従来通り維持する。**削除するのはラベルテキストのみ**で、操作制御は変更しない。

#### ③ 過去週の案内テキストの統計画面へのリンク化
- 過去の週のデータ表示時の「**過去の記録を閲覧できます**」というテキストを、**クリッカブルなテキストリンク（またはボタン）**に変更する。
- 押下時に `/statistics`（統計画面）へ遷移するルーティングを設定する。
- リンクであることが視覚的に分かるスタイル（下線 / プライマリカラー等）を付与する。

#### ④ 「モーニングページ」の「未完了」ラベル非表示
- 「モーニングページ」に表示していた「**未完了**」というテキストを**削除**する。
- ※ 達成/未達成の状態はドットグリッドの色や「X/7」表記（v2.3）で表現済みのため、冗長な「未完了」ラベルは表示しない。

#### ⑤ 達成時セレブレーション演出
- **発火条件（OR・いずれか一方を満たせば発火）**:
  1. 「モーニングページ」の今週の進捗が **7/7（コンプリート）** になった。
  2. 「アーティストデート」が**完了（実施済み）**になった。
- **演出スタイル（本バージョンの方針：控えめ）**:
  - 該当条件を満たした**項目のカード単位**で演出を適用する（モーニングページのコンプリートはモーニングページのカード、アーティストデートの完了はアーティストデートのカードに適用。両方達成時は両カードに適用）。
  - **背景色の変化**: 該当カードの背景を特別感のある色（薄いゴールド / ソフトなグラデーション）に変化させる。
  - **アイコンの変更**: チェックマークを豪華なアイコン（王冠 👑 / キラキラ ✨ / クラッカー 🎉 など）に変更する。
  - 派手なアニメーション（紙吹雪・パーティクル等）は本バージョンでは採用せず、静的な祝福スタイル（背景＋アイコン）を基本とする。
- **適用タイミング（本バージョンの方針：状態ベース・毎回適用）**:
  - 「瞬間の検出」は行わず、**達成条件を満たしている限り、ホームを表示するたびに**祝福スタイルを適用する（リロード・再訪のたびに表示される）。
  - 軽微なワンポイントの動き（アイコンのポップ等）を添える場合は `prefers-reduced-motion` を尊重し、モーション低減設定時は静的表示にとどめる。
- **判定データ**: `GET /api/progress` の `morning_pages_this_week === 7`（または `daily_status` の `done` が7件）、`artist_date_done === true`（または `artist_date.went_out` / `excited` の方針に従う）。
  - ※ 「アーティストデート完了」を `went_out` のみで判定するか、`went_out && excited` で判定するかは実装時に確定する（既定は「`went_out` が true なら完了」とする）。

#### レイアウト（参考）
```
   ＜  [第N週目]  ＞   [現在の週に戻る]

┌────────────────────────────────────────┐
│ ✨ モーニングページ (今週の実行状況) 7/7 │ ← 7/7達成: 背景=薄ゴールド
│ ●●●●●●●  👑                         │    アイコン=豪華アイコン
│ Day1 .. [Day4(今日)] .. Day7           │ ← 本日: バッジ「今日」+ リング強調
└────────────────────────────────────────┘
   ※ 未達成時は通常背景・通常アイコン。「未完了」ラベルは表示しない。

┌──────────────────┐ ┌──────────────────┐
│ モーニングページ→ │ │ 🎉アーティスト    │ ← 実施済み: 背景=薄ゴールド
│                  │ │  デート (完了)→   │    アイコン=クラッカー
└──────────────────┘ └──────────────────┘

(過去週を閲覧中)
┌────────────────────────────────────────┐
│ 第2週目（過去）                         │ ← 「(閲覧のみ)」テキストは削除
│ [過去の記録を閲覧できます]              │ ← クリックで /statistics へ遷移(リンク化)
└────────────────────────────────────────┘
```

## 3. 画面遷移図（テキスト表現・v2.4 差分）
