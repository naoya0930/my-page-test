# spec4: UI/UX改善および機能追加仕様書

## 1. アカウント登録フローの改修
* **利用規約の同意必須化**: 
    * サインアップ（アカウント作成）画面に「利用規約およびプライバシーポリシーに同意する」チェックボックスを追加する。
    * チェックボックスが未チェックの状態では、登録ボタン（Submit）を非活性（disabled）にする、または押下時にバリデーションエラーメッセージを表示して処理をブロックする。

## 2. プログラム完遂時の演出追加
* **12週間完了時の祝福ポップアップ**:
    * ユーザーが第12週の最終日を終えた（あるいは12週分のProgressを満たした）とシステムが判定した際、ログイン直後やホーム画面遷移時に専用のモーダル（ポップアップ）を表示する。
    * 「12週間のプログラム完遂、おめでとうございます！」といった労いのメッセージと、可能であれば紙吹雪（Confetti）などの視覚的な演出を取り入れ、達成感を高める。

## 3. `/home`（ホーム画面）のUI改善
* **現在日数の明記**:
    * 画面の上部またはヘッダー付近に、プログラム開始日を起点とした現在の経過日数を「**本日: Day X**」（例: Day 15）と明確に表示する。
* **モーニングページ進捗の表記変更**:
    * 現在の週におけるモーニングページの達成状況を、直感的な「**X/7**」という分母・分子のテキスト形式で表示する（例：今週3日書いた場合は「3/7」）。
* **ナビゲーションの視認性向上**:
    * **週移動ボタン（`<` / `>`）**: 可視性を広げる。
    * **「現在の週に戻る」ボタン**: 過去の週を閲覧している際にのみ表示し、プライマリカラー（目立つ色）を使用する。
* **保存完了のフィードバック（トースト通知）**:
    * `/morning-page` 画面で「保存する」を実行し、`/home` にリダイレクトされた直後に、「モーニングページを保存しました」という一時的な通知（トースト/スナックバー）を画面下部または上部に数秒間表示し、フェードアウトさせる。

## 4. `/statistics`（統計画面）のUI改善
* **日次データの選択と詳細確認**:
    * カレンダーやリスト表示されている各日の実績を、ラジオボタンのように単一選択（アクティブ状態を切り替え可能）できるUIにする。
    * 日付を選択すると「**詳細**」ボタン（またはリンク）が出現し、押下することで該当日のモーニングページ内容（閲覧・編集用モーダル、または専用URL）を展開できるようにする。
* **アーティストデート軌跡の可視化**:
    * 各週のアーティストデート実施状況（リストまたはグラフの軸）において、結果を直感的な記号で表現する。
    * 実施した週は「**〇**（マル / アクセントカラー）」、未実施の週は「**✕**（バツ / グレー）」のアイコンまたはテキストを配置し、ひと目で達成状況が把握できるようにする。
