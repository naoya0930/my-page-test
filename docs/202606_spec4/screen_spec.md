## 1. 画面一覧・ルーティング

| パス | 画面名 | アクセス制限 | 概要 |
| :--- | :--- | :--- | :--- |

## 2. 各画面詳細仕様（v2.3 追加・変更分）

### 2.1 ログイン・アカウント作成画面（改修）
- **パス**: `/login`
- **追加要件: 利用規約の同意必須化**
  - サインアップ（アカウント作成）モード時に「**利用規約およびプライバシーポリシーに同意する**」チェックボックスを追加する。
  - チェックボックスが未チェックの状態では、登録ボタン（「アカウントを作成する」）を**非活性（disabled）**にする。
  - あわせて、未チェックのまま押下された場合は「利用規約への同意が必要です」のバリデーションエラーを表示し、処理をブロックする（二重防御）。
  - ログインモードではチェックボックスを表示しない。
- **挙動・仕様**:
  - 認証基盤は `Supabase Auth` を利用（既存仕様を踏襲）。
  - クライアントバリデーション（メール形式、パスワード一致、**利用規約同意**）の通過後にサインアップAPIを呼び出す。

#### レイアウト（参考）
```
┌─────────────────────────────────────┐
│ [ ログイン ] [ 新規アカウント作成 ]   │ ← タブ切り替え
├─────────────────────────────────────┤
│ メールアドレス [____________]        │
│ パスワード     [____________]        │
│ パスワード(確認)[____________]        │
│ ☐ 利用規約およびプライバシーポリシー  │
│    に同意する                        │
│ [エラーメッセージ] (条件表示)        │
│ [ アカウントを作成する ]             │ ← 同意チェックまで非活性
└─────────────────────────────────────┘
```

### 2.2 ユーザーホーム画面（改修）
- **パス**: `/home`

#### ① 現在日数の明記
- 画面の上部またはヘッダー付近に、プログラム開始日を起点とした現在の経過日数を「**本日: Day X**」（例: Day 15）と明確に表示する。
- 既存の「Day XX / 84」表記に「本日:」ラベルを付与し、「**本日: Day X / 84**」として表示する。
- 表示週の切り替え（タイムトラベル）とは独立して、常に本人の経過日数を基準に算出する。

#### ② モーニングページ進捗の表記変更
- 現在の週におけるモーニングページの達成状況を、直感的な「**X/7**」という分母・分子のテキスト形式で表示する（例：今週3日書いた場合は「3/7」）。
- 7日間のドットグリッドの近傍（見出し横など）に併記する。

#### ③ ナビゲーションの視認性向上
- **週移動ボタン（`<` / `>`）**: クリック/タップ領域とサイズを広げ、可視性を高める。
- **「現在の週に戻る」ボタン**: 過去/未来の週を閲覧している際**にのみ**表示し、プライマリカラー（目立つ色）を使用する。押下で現在の経過週へ戻る。現在週の表示中は非表示。

#### ④ 保存完了のフィードバック（トースト通知）
- `/morning-page` で「保存する」を実行し、`/home` にリダイレクトされた直後に、「**モーニングページを保存しました**」という一時的な通知（トースト/スナックバー）を画面下部または上部に数秒間表示し、フェードアウトさせる。
- リダイレクト時にナビゲーション state 等で保存成功を `/home` へ伝達し、リロード時に再表示されないようにする。

#### ⑤ プログラム完遂時の祝福ポップアップ
- ユーザーが第12週の最終日を終えた（**経過日数が Day 84 に到達した**）とシステムが判定した際、ログイン直後やホーム画面遷移時に専用のモーダル（ポップアップ）を表示する。
- 「**12週間のプログラム完遂、おめでとうございます！**」といった労いのメッセージと、紙吹雪（Confetti）などの視覚的な演出で達成感を高める。
- **判定基準**: `GET /api/progress` の `current_day >= 84`。
- **再表示制御（本バージョンの方針）**: 到達後はログインのたびに表示する（表示済みフラグの保存は行わない）。
- 「閉じる」操作でホーム画面に戻れること。`prefers-reduced-motion` 設定時は演出を控えめにする。

#### レイアウト（参考）
```
┌────────────────────────────────────────┐
│           本日: Day 17 / 84             │ ← 現在日数の明記
└────────────────────────────────────────┘

   ＜  [第N週目]  ＞   [現在の週に戻る]
   (「現在の週に戻る」は過去/未来週の閲覧中のみ表示)

┌────────────────────────────────────────┐
│ モーニングページ (今週の実行状況)  3/7  │ ← X/7 テキスト併記
│ ●●●○○○○                            │
└────────────────────────────────────────┘

  ┌──────────────────────────────────┐
  │ ✓ モーニングページを保存しました   │ ← トースト(数秒で消える)
  └──────────────────────────────────┘

  ※ Day 84 到達時は祝福モーダル(紙吹雪)を重畳表示
  ┌──────────────────────────────────┐
  │   🎉 12週間のプログラム完遂、      │
  │      おめでとうございます！        │
  │            [ 閉じる ]             │
  └──────────────────────────────────┘
```

### 2.3 統計画面（改修）
- **パス**: `/statistics`

#### ① 日次データの選択と詳細確認
- カレンダーやリスト表示されている各日の実績を、**ラジオボタンのように単一選択**（アクティブ状態を切り替え可能）できるUIにする。1つ選択すると以前の選択は解除される。
- 日付を選択すると「**詳細**」ボタン（またはリンク）が出現し、押下することで該当日のモーニングページ内容を確認できる。
- **詳細の表示方法（本バージョンの方針）**: 既存の `/morning-page?date=YYYY-MM-DD` 画面へ遷移して閲覧・編集する（spec3 で実装済みの日付指定モーニングページ画面を再利用）。過去日の場合はモーニングページ画面側のルール（過去ページ編集の確認・保存後 `/home` リダイレクト）に従う。

#### ② アーティストデート軌跡の可視化
- 各週のアーティストデート実施状況（リストまたはグラフの軸）において、結果を直感的な記号で表現する。
- **実施した週**: 「**〇**（マル / アクセントカラー）」。
- **未実施の週**: 「**✕**（バツ / グレー）」。
- `went_out` / `excited` をそれぞれ 〇/✕ で表現し、ひと目で達成状況が把握できるようにする。

#### レイアウト（参考）
```
┌────────────────────────────────────────┐
│ 日次アクティビティ（選択可）             │
│ [選択中: 6/12] → [ 詳細 ]               │ ← /morning-page?date= へ遷移
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│ アーティストデート軌跡                  │
│ Week1: 〇出 〇わく                      │
│ Week2: 〇出 ✕わく                      │
│ ... (実施=〇/アクセント, 未実施=✕/グレー)│
└────────────────────────────────────────┘
```

## 3. 画面遷移図（テキスト表現・v2.3 差分）
```
ログイン・アカウント作成画面 (/login)
└── [新規アカウント作成] ──> 利用規約同意チェック(必須) ──> 登録成功 ──> ユーザーホーム (/home)
        ※ 未同意の間は登録ボタンを非活性

ユーザーホーム画面 (/home)
│   ※ 上部に「本日: Day X / 84」、モーニングページ進捗「X/7」を表示
│   ※ Day 84 到達時はログイン直後/遷移時に祝福モーダル(紙吹雪)を表示
├── [現在の週に戻る] (過去/未来週の閲覧中のみ表示) ──> 現在の経過週へ
└── (モーニングページ保存後の遷移直後にトースト通知を表示)

モーニングページ画面 (/morning-page)
└── [保存する] ──> 成功 ──> (自動リダイレクト) ──> /home でトースト「保存しました」

統計画面 (/statistics)
├── [日付を単一選択 ──> 詳細] ──> モーニングページ画面 (/morning-page?date=YYYY-MM-DD)
└── アーティストデート軌跡を 〇(実施)/✕(未実施) で表示
```
